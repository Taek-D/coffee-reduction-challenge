# 커밋 → 푸시 → PR 생성

변경 사항을 커밋하고 원격에 푸시한 뒤 PR을 생성합니다.

## 절차

1. `npm run typecheck` 실행하여 타입 오류 확인
2. `npm run test` 실행하여 테스트 통과 확인
3. `npm run lint` 실행하여 린트 오류 확인
4. `git status`로 변경 파일 확인
5. `git diff --staged`로 스테이징된 변경 확인
6. 변경 사항을 분석하여 커밋 메시지 작성 (한국어, 간결하게)
7. 커밋 생성
8. 원격 브랜치에 푸시
9. `gh pr create`로 PR 생성 (제목: 한국어, 본문: 변경 요약 + 테스트 계획)

## 규칙

- 타입체크/테스트/린트 중 하나라도 실패하면 커밋하지 않는다
- 커밋 단위: 화면 1개 또는 레이어 1개
- `.env`, `*.local` 파일은 커밋하지 않는다
- PR 대상 브랜치: `main`
