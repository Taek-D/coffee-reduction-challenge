# 출시 전 종합 점검 설계

Date: 2026-03-03
Status: Approved
Approach: A (앱인토스 출시 체크리스트 중심 종합 점검)

## 목적

앱인토스 비게임 출시 검수 반려를 예방하고, 코드 품질/성능/정책 준수를 종합적으로 확인한다.
결과는 보고서로 정리하며, 코드 수정은 별도 세션에서 진행한다.

## 점검 영역 (7개)

### 1. 비게임 출시 체크리스트 (15항목)

PRD 섹션 6의 비게임 출시 체크리스트를 코드/설정에 매핑하여 PASS/FAIL/WARNING 판정.

| # | 항목 | 점검 방법 |
|---|------|----------|
| 1 | 시스템 모드: 라이트 모드 고정 | index.html, index.css 확인 |
| 2 | 확대/축소: 핀치줌 비활성화 | meta viewport 확인 |
| 3 | 내비게이션 바: 브랜드 로고+이름 | granite.config.ts, SDK 설정 확인 |
| 4 | 전반 서비스: 나이제한, 2초 반응, 재접속 유지, 컴포넌트 | 번들 크기, Storage 로직 확인 |
| 5 | 접근성: 명도 대비, 터치 영역, 스크린 리더 | CSS, 컴포넌트 속성 확인 |
| 6 | 앱 내 기능: 딥링크, 뒤로가기→메인 | routes.tsx, 네비게이션 로직 확인 |
| 7 | 토스 로그인: 약관, 로그인 끊기, 인트로 | LoginPage, AppContext 확인 |
| 8 | 토스페이: 해당 없음 | N/A |
| 9 | 인앱결제: 금액, 결제/취소, 에러, 내역, 기기변경 | IAP 코드 전체 추적 |
| 10 | 프로모션: 현금성/환금성 이벤트 금지 | 코드 스캔 |
| 11 | 기능성 메시지: MVP 미사용 | N/A |
| 12 | 인앱광고: MVP 미사용 | 코드 스캔 (광고 SDK 없음 확인) |
| 13 | 앱 사용 권한 | permissions 배열 확인 |
| 14 | 데이터/메모리 | 번들 크기, 메모리 누수 패턴 확인 |
| 15 | 보안 | 개인정보 처리, 민감 데이터 확인 |

### 2. 카피/문구 정책 스캔

src/ 전체에서 다음 금지 패턴을 grep으로 검색:
- 건강/의학: 치료, 진단, 개선, 중독, 건강, 카페인 효과
- 보장 표현: 보장, 확실, 반드시, 틀림없이
- 자동연동 오해: 통장, 잔액, 계좌, 카드 내역
- 외부 링크: http://, https:// (허용 목록 제외)
- 밈/유행어

### 3. 다크패턴 방지 정책 (5항목)

| # | 정책 | 점검 |
|---|------|------|
| 1 | 진입 시 바텀시트 차단 금지 | 온보딩/메인 진입 코드 확인 |
| 2 | 뒤로가기 시 인터럽트 금지 | 라우팅 로직 확인 |
| 3 | 나갈 수 있는 선택지 제공 | 프리미엄 화면 CTA 확인 |
| 4 | 예상치 못한 광고 금지 | 광고 관련 코드 부재 확인 |
| 5 | CTA 명확성 | 모든 버튼 텍스트 확인 |

### 4. 코드 품질 자동 검증

```bash
npm run typecheck
npm run lint
npm run test
npm run build
```

각 명령 실행 결과를 기록.

### 5. 성능/번들 예산

- dist/ 파일 크기 (raw + gzip)
- 이미지/폰트 외부 로드 여부
- 불필요한 의존성 확인

### 6. IAP/결제 코드 리뷰

구매 → 지급 → 복원 → 만료 플로우를 코드 레벨에서 추적:
- `getIapProductList` → 상품 목록 정합성
- `createIapOrder` → 에러 핸들링
- `completeIapProductGrant` → 지급 완료 처리
- `getPendingIapOrders` → 미결 주문 복원
- `PremiumStatus` → 만료 체크 로직

### 7. 빈상태/에러/로딩 커버리지

모든 페이지 컴포넌트에서:
- 로딩 상태 처리 여부
- 빈상태(데이터 없음) 처리 여부
- 에러 상태 처리 여부
- Toast 메시지 PRD 일치 여부
