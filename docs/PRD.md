# 커피 줄이기 챌린지 PRD

이 문서는 현재 출시 가능한 MVP 범위를 빠르게 확인하기 위한 요약본이에요. 상세 정의와 화면/정책/QA 기준은 [../커피-줄이기-챌린지-PRD.md](../커피-%EC%A4%84%EC%9D%B4%EA%B8%B0-%EC%B1%8C%EB%A6%B0%EC%A7%80-PRD.md)를 기준으로 해요.

## 현재 확정한 MVP 범위

- 오늘 커피 수 기록, 기준선 설정, 목표 설정
- 월 요약 중심의 절감액/지출 확인
- 프리미엄 PDF 저장과 기간제 소모성 IAP
- Storage SDK 우선, 브라우저 fallback 기반의 동일 기기 저장
- Apps-in-Toss 런타임에서 `appLogin` 진입 시도

## MVP에서 제외한 범위

- 재설치/기기 변경 시 데이터 자동 복원
- 완료 주문 이력 기반 구매 복원
- 목표 달성률, 심화 패턴 인사이트, 과한 게이미피케이션
- 토스 로그인 자동 userKey 획득을 모든 환경에서 필수로 보장하는 요구

## 현실화한 이유

현재 구현은 로컬 저장소와 SDK 기본 흐름을 중심으로 구성돼 있어요. 서버 동기화 계층 없이 `동일 userKey 로그인만으로 새 기기 복원`을 보장할 수는 없기 때문에, 그 요구는 Phase 2로 이동했어요.

또한 홈 핵심 루프는 `기록 -> 이번 달 절감 확인`까지는 이미 맞출 수 있지만, 목표 달성률과 정교한 인사이트는 현재 구현과 2주 MVP 일정 기준에서 우선순위를 낮췄어요.

## 운영 기준

- 로그인은 선택 진입이에요. 로그인에 실패하거나 자동 식별이 불가능해도 현재 기기에서 사용할 식별값으로 계속 사용할 수 있어요.
- 데이터와 프리미엄 상태는 동일 기기 기준으로만 유지돼요.
- 설정의 `구매 복원`은 완료 주문 전체 복원이 아니라 미결 주문 재처리 의미로만 사용해요.
- 앱 소개/설정/심사 대응 문구에도 위 제한을 그대로 반영해요.
